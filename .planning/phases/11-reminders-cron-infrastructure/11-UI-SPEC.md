## Interaction Spec

### FollowUpEditor (application detail view)

**States:**
1. **Display (no follow-up):** Shows "No follow-up set" with "Set follow-up" button.
2. **Display (follow-up set):** Shows "Follow-up: May 15" with "Edit" and "Clear" buttons.
3. **Editing:** Shows quick-set buttons ("Tomorrow", "Next week", "Next month") + date input + "Save" / "Cancel".

**Interactions:**
- Click "Set follow-up" or "Edit" -> transitions to editing state.
- Click quick-set button -> calculates date, saves immediately, returns to display state.
- Select date from picker + click "Save" -> saves, returns to display state.
- Click "Clear" -> removes follow-up date, returns to display state (no confirmation dialog — clear is non-destructive, date can be re-set).
- Click "Cancel" -> returns to display state without saving.

**Placement:** Between StageEditor and NotesEditor in ApplicationDetail (D-01).

### FollowUpsCard (Today dashboard)

**States:**
1. **Has overdue follow-ups:** Lists cards sorted oldest-first (D-06). Each card shows role title, company, days overdue (D-07). Clicking navigates to application detail.
2. **Empty:** Shows empty state copy per copywriting contract.

**Placement:** New section on Today dashboard using `TodaySection` wrapper (D-08).

### Count badge (Today nav link)

**Behavior:**
- Badge appears only when count > 0 (D-09).
- Badge shows numeric count in a pill (D-11).
- Badge uses primary/accent color (`#d97706` light, `#f59e0b` dark) — consistent with brand accent.
- Badge disappears when no overdue follow-ups (D-09).

**Implementation note:** `NavLinks` is currently a client component (`'use client'`). The count must be fetched server-side and passed as a prop, or the Today page must pass the count via a shared state mechanism. Prefer server-component prop drilling from layout.

## Accessibility

- Quick-set buttons: keyboard-focusable, `aria-label` with computed date (e.g., "Set follow-up to May 11, 2026").
- Date input: native `<input type="date">` for built-in keyboard/screen-reader support.
- Count badge: `aria-label="X follow-ups due"` on the Today nav link.
- Follow-up cards in Today: each card is a `<Link>` (already keyboard-focusable), `aria-label` includes role + company + days overdue.
- Clear button: `aria-label="Clear follow-up date"`.
