The challenge: NavLinks is a `'use client'` component rendered in the root layout. The overdue count must be fetched server-side. The solution is a server wrapper component.

**Step 1: Create a server wrapper component**

Create `src/components/nav-links-wrapper.tsx` (server component, no `'use client'`):

```typescript
import { requireUser } from '@/core/auth/session'
import { findOverdueFollowUps } from '@/features/today/queries'

import { NavLinks } from './nav-links'

export async function NavLinksWrapper() {
  let overdueCount = 0

  const userResult = await requireUser()
  if (userResult.isOk()) {
    const result = await findOverdueFollowUps(userResult.value.id)
    if (result.isOk()) {
      overdueCount = result.value.length
    }
  }

  return <NavLinks overdueCount={overdueCount} />
}
```

**Step 2: Modify NavLinks to accept overdueCount prop**

In `src/components/nav-links.tsx`:

1. Add `overdueCount` prop to the component signature:
```typescript
export function NavLinks({ overdueCount = 0 }: { overdueCount?: number })
```

2. Modify the Today link rendering to include a badge when `overdueCount > 0`. The badge is a small pill with the count number, using accent color (per D-09, D-11):

```tsx
{links.map(({ href, key }) => {
  const active = pathname === href || pathname.startsWith(`${href}/`)
  const showBadge = href === '/today' && overdueCount > 0

  return (
    <Link
      key={href}
      href={href}
      aria-current={active ? 'page' : undefined}
      aria-label={showBadge ? `${t(key)} (${overdueCount} follow-ups due)` : undefined}
      className={cn(
        'rounded-md px-3 py-2 text-sm transition',
        active
          ? 'bg-accent font-medium text-accent-foreground'
          : 'text-muted-foreground hover:bg-accent/50 hover:text-foreground',
      )}
    >
      {t(key)}
      {showBadge && (
        <span className="ml-1.5 inline-flex items-center justify-center rounded-full bg-primary px-1.5 py-0.5 text-[10px] font-medium text-primary-foreground">
          {overdueCount}
        </span>
      )}
    </Link>
  )
})}
```

The badge uses `bg-primary` (accent color per DESIGN.md) and `text-primary-foreground` for contrast. The `text-[10px]` size keeps it compact as a pill.
