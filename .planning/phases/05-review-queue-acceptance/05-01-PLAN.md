  <action>
    Create `src/app/api/inbox/full-body/route.ts` ‚Äî a Route Handler that fetches full email body from Gmail API with in-memory token bucket rate limiting.

    **Rate limiter** (in-memory, single-user sufficient):
    ```typescript
    const buckets = new Map<string, { tokens: number; lastRefill: number }>()
    const MAX_TOKENS = 5
    const REFILL_RATE = 5 // tokens per second

    function consumeToken(userId: string): boolean {
      const now = Date.now()
      const bucket = buckets.get(userId) ?? { tokens: MAX_TOKENS, lastRefill: now }
      const elapsed = (now - bucket.lastRefill) / 1000
      bucket.tokens = Math.min(MAX_TOKENS, bucket.tokens + elapsed * REFILL_RATE)
      bucket.lastRefill = now
      if (bucket.tokens < 1) { buckets.set(userId, bucket); return false }
      bucket.tokens -= 1
      buckets.set(userId, bucket)
      return true
    }
    ```

    **GET handler**:
    1. `requireUser()` ‚Äî return 401 JSON `{ error: 'Unauthorized' }` on failure
    2. `consumeToken(userId)` ‚Äî return 429 JSON `{ error: 'Rate limited' }` with `Retry-After: 1` header on failure
    3. Read `?messageId=` from `request.nextUrl.searchParams` ‚Äî return 400 if missing
    4. `getGmailClient(UserId(userId))` ‚Äî return 503 JSON `{ error: 'Gmail disconnected ‚Äî full bodies unavailable' }` on failure
    5. `gmail.users.messages.get({ userId: 'me', id: gmailMessageId, format: 'full' })`
    6. Extract plain text body using the same `extractPlainTextBody` pattern from `gmail-client.ts` (inline the recursive part extraction ‚Äî do NOT import it, it's a private function)
    7. Return 200 JSON `{ body: string }`

    Import `requireUser` from `@/core/auth/session`, `getGmailClient` from `@/features/inbox/gmail-client`, `UserId` from `@/core/types/ids`.

    Do NOT create a test file for this route handler ‚Äî the rate limiter is tested implicitly via the browser verify checkpoint in Plan 02, and the Gmail client is already tested in `gmail-oauth.test.ts`.

| Boundary | Description |
|----------|-------------|
| client -> Server Actions | User-submitted emailId/applicationId crosses here |
| client -> Route Handler | Query params (messageId) cross here |
| Route Handler -> Gmail API | External API call with stored OAuth token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-05-01 | Tampering | Server Actions (review actions) | mitigate | `requireUser()` in every action; `withRls` enforces tenant scope ‚Äî user cannot review another user's emails |
| T-05-02 | Information Disclosure | /api/inbox/full-body | mitigate | `requireUser()` + rate limit (5/sec) prevents Gmail API abuse; no email body stored in DB (excerpt only) |
| T-05-03 | Denial of Service | /api/inbox/full-body | mitigate | Token bucket at 5 req/sec/user; 429 with Retry-After header |
| T-05-04 | Elevation of Privilege | linkToApplication action | mitigate | ApplicationId validated via Zod schema; `withRls` ensures user can only link to their own applications |
</threat_model>
