    - If null, render `—` (dash) in muted text

    - Render 3 `<div>` bars (h-3 w-1.5 rounded-sm), colored by tier
    - Exact percentage on hover via CSS tooltip: `<span>` with `opacity-0 group-hover:opacity-100` positioned above the bars
    - Use Tailwind colors: `bg-green-500` (high), `bg-amber-500` (medium), `bg-gray-400` (low), `bg-gray-200` (inactive bar)

    **2. `classification-select.tsx`** — `'use client'` component:
    - Props: `{ emailId: number; currentClassification: EmailClassification | null; onOverride: (emailId: number, newClassification: EmailClassification) => void }`
    - Use shadcn `Select` component with all 5 EmailClassification options as values
    - On value change, call `onOverride(emailId, newValue)`
    - Display current classification as selected value

    **3. `link-application-dialog.tsx`** — `'use client'` component:
    - Props: `{ emailId: number; applications: Array<{id: number, roleTitle: string, companyName: string}>; onLink: (emailId: number, applicationId: number) => void }`
    - Use shadcn `Dialog` with a trigger button ("Link to application")
    - Inside dialog: searchable input filtering `applications` list by roleTitle or companyName
    - List matching applications as clickable rows showing `{roleTitle} — {companyName}`
    - On select, call `onLink(emailId, selectedAppId)` and close dialog
    - Use `useState` for search query filtering

    **4. `degradation-banner.tsx`** — Server Component (no `'use client'`):
    - Props: `{ gmailConnected: boolean; lastSyncAt: Date | null }`
    - If `!gmailConnected`: render warning banner "Gmail disconnected — full bodies unavailable. Connect Gmail in Settings."
    - If `lastSyncAt` is null OR `Date.now() - lastSyncAt.getTime() > 5 * 24 * 60 * 60 * 1000` (5 days): render warning "Last sync was over 5 days ago. Gmail Test-mode tokens expire after 7 days."
    - Style: amber/yellow background banner at top of inbox, text-sm, with link to `/settings`
    - If neither condition: render nothing (return null)

    **5. `inbox-row.tsx`** — `'use client'` component:
    - Props: `{ item: InboxItem; applications: Array<{id: number, roleTitle: string, companyName: string}>; onConfirm: (emailId: number) => void; onOverride: (emailId: number, classification: EmailClassification) => void; onLink: (emailId: number, applicationId: number) => void; onIgnore: (emailId: number) => void }`
    - Layout: Card with subject as header, from + receivedAt in subtitle
    - Body excerpt: first 200 chars in muted text, truncated with "..."
    - Classification badge: colored badge showing the label (rejection=red, interview=green, outreach=blue, noise=gray, unmatched=yellow) — use existing shadcn Badge with variant mapping
    - Confidence: `<ConfidenceBadge confidence={item.confidence} />`
    - Suggested application: if `applicationId` is set, show `{applicationRoleTitle} at {companyName}` as a link to `/applications/{applicationId}`
    - Actions row: 4 buttons — "Confirm" (primary), "Override" (renders `<ClassificationSelect>`), "Link" (renders `<LinkApplicationDialog>`), "Ignore" (ghost/destructive)
    - On confirm click: call `onConfirm(item.id)` — parent will remove from list
    - On ignore click: call `onIgnore(item.id)` — parent will remove from list

    **6. `inbox-list.tsx`** — `'use client'` component:
    - Props: `{ items: InboxItem[]; applications: Array<{id: number, roleTitle: string, companyName: string}> }`
    - Maintains local state `const [emails, setEmails] = useState(items)`
    - Action handlers that call the Server Actions from `@/features/inbox/actions`:
      - `handleConfirm(emailId)`: call `confirmClassification(emailId)`, if ok remove from local state
      - `handleOverride(emailId, classification)`: call `overrideClassification(emailId, classification)`, if ok remove from local state
      - `handleLink(emailId, applicationId)`: call `linkToApplication(emailId, applicationId)`, if ok remove from local state
      - `handleIgnore(emailId)`: call `ignoreEmail(emailId)`, if ok remove from local state
    - If `emails.length === 0`: render empty state "No emails to review. All caught up."
    - Otherwise: map `emails` to `<InboxRow>` components
    - Note: Server Actions are called directly (not via `useTransition`) — the optimistic removal from local state provides instant feedback. Call `revalidatePath` inside the actions (already done in Plan 05-01) to sync server state.

    Per DESIGN.md: text labels over icons, minimal decorative elements, functional icons only (close, nav arrows). No emoji. Neutral palette with status-colored accents.
