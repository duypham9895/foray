    1. **Create `extension/entrypoints/popup/main.ts`** — the popup's main logic:

       **On popup open:**
       - Read `apiToken` and `forayUrl` from `chrome.storage.local`
       - If no token: show setup hint ("Set your API token in extension options"), disable Capture button
       - If token exists: use `chrome.tabs.query({ active: true, currentWindow: true })` to get current tab
       - Display tab title and URL in the popup
       - Enable Capture button

       **On Capture click:**
       - Disable button, show "Capturing..." state
       - Read `forayUrl` from storage (default: `http://localhost:3000`)
       - POST to `${forayUrl}/api/capture` with:
         ```typescript
         fetch(`${forayUrl}/api/capture`, {
           method: 'POST',
           headers: {
             'Content-Type': 'application/json',
             'Authorization': `Bearer ${token}`,
           },
           body: JSON.stringify({ title: tab.title, url: tab.url }),
         })
         ```
       - On success (200): parse `{ redirectUrl }`, show success status ("Captured. Open in Foray to continue."), show "Open in Foray" link pointing to `${forayUrl}${redirectUrl}`
       - On 401: show error "Invalid token. Check extension settings." + link to options page
       - On other errors: show error message from response
       - Store `lastCapture: { title, url, timestamp }` in `chrome.storage.local` per EXT-07

       **"Open in Foray" link:**
       - Opens `${forayUrl}${redirectUrl}` in a new tab
       - This takes the user to `/applications/new?prefilled=<base64>` with the captured data pre-filled

    2. **Token storage in chrome.storage.local** per EXT-07:
       - Shape: `{ apiToken: string, forayUrl: string, lastCapture?: { title: string, url: string, timestamp: string } }`
       - NO module-level variables — all state read from/written to chrome.storage.local
       - This ensures state survives MV3 service worker restarts (30s timeout)

    3. **Add options page** (simple inline page for token configuration):
       - In `wxt.config.ts`, add `options_ui` to manifest:
         ```typescript
         manifest: {
           // ...existing
           options_ui: {
             page: 'options/index.html',
             open_in_tab: false,
           },
         }
         ```
       - Create `extension/entrypoints/options/index.html` — simple form with:
         - Input for Foray URL (default: http://localhost:3000)
         - Input for API Token
         - Save button
         - On save: store to `chrome.storage.local`
       - This is where the user pastes the token generated in Plan 01's settings UI

    4. **Popup size and feel** per DESIGN.md:
       - Width: 320px, min-height: 200px
       - Padding: 16px
       - H1 "Foray" in `text-lg font-medium` (not `text-3xl` — popup is small)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| extension popup -> chrome.storage.local | Token stored locally in browser |
| extension popup -> capture API | Bearer token sent over network |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-05 | Information Disclosure | chrome.storage.local | accept | Single-user local deployment; token is for local dev server. Production would use encrypted storage. |
| T-13-06 | Spoofing | popup -> API | mitigate | Bearer token verified server-side (Plan 01); HTTPS not required for localhost dev |
| T-13-07 | Tampering | capture payload | mitigate | Server-side URL validation + ATS domain rejection already in capture route |
</threat_model>
