    2. **Update `extension/entrypoints/background.ts`** — SPA navigation detection:
       ```typescript
       export default defineBackground(() => {
         // Detect SPA navigation (pushState/replaceState) — LinkedIn, Greenhouse, Lever, etc.
         chrome.webNavigation.onHistoryStateUpdated.addListener(
           (details) => {
             // Only respond to top-level frame navigations
             if (details.frameId !== 0) return;

             // Notify the content script that the URL changed
             chrome.tabs.sendMessage(details.tabId, {
               type: 'URL_CHANGED',
               url: details.url,
             }).catch(() => {
               // Content script may not be injected yet — safe to ignore
             });
           },
           // Only monitor http/https URLs
           { url: [{ schemes: ['http', 'https'] }] }
         );

         console.log('Foray background: SPA navigation listener registered');
       });
       ```
       Per EXT-07: no module-level state. The listener is stateless — it just forwards URL changes to the content script.

    3. **Update `extension/entrypoints/content.ts`** — listen for URL changes and re-extract page info:
       ```typescript
       export default defineContentScript({
         matches: ['<all_urls>'],
         main() {
           // Listen for URL change messages from background script
           chrome.runtime.onMessage.addListener((message, sender, sendResponse) => {
             if (message.type === 'URL_CHANGED') {
               // The URL changed via SPA navigation — update stored page info
               // The popup will read this fresh on next open
               chrome.storage.local.set({
                 lastPageInfo: {
                   title: document.title,
                   url: message.url || window.location.href,
                   timestamp: new Date().toISOString(),
                 },
               });
             }
           });

           // Also store initial page info on script injection
           chrome.storage.local.set({
             lastPageInfo: {
               title: document.title,
               url: window.location.href,
               timestamp: new Date().toISOString(),
             },
           });
         },
       });
       ```
       Per EXT-07: page info stored in chrome.storage.local, not module variables.

    4. **Update `extension/entrypoints/popup/main.ts`** — read from storage as fallback:
       - When popup opens, try `chrome.tabs.query` first (most accurate)
       - If tab query returns stale data (e.g., SPA navigated but popup cached), also check `chrome.storage.local.lastPageInfo`
       - Prefer tab query if available; fall back to stored info
       - This handles the edge case where the service worker woke up after 30s and the content script already stored updated info
