Props interface:
```typescript
type DocumentRow = {
  id: number

  sizeBytes: number
  createdAt: Date
}

type Props = {
  documents: DocumentRow[]
  applicationId: number
}
```

Implementation:
1. Render a list with each document showing: Kind (badge), Filename (download link), Size (formatted), Uploaded (formatted date), Actions (delete button)
2. Kind display: map DocumentKind to human-readable label — resume -> "Resume", cover_letter -> "Cover Letter", jd_pdf -> "JD PDF", take_home -> "Take-home", other -> "Other". Use a small colored badge matching the tag-editor style.
3. Size formatting: inline helper `formatSize(bytes)` — show KB for < 1024, MB for >= 1024*1024, e.g., "245 KB", "1.2 MB"
4. Filename: render as `<a href="/api/documents/{id}" target="_blank" download className="underline ...">{filename}</a>` — browser handles download via Content-Disposition header
5. Delete button: onClick calls handler that:
   - Shows `window.confirm('Delete {filename}? This cannot be undone.')`
   - On confirm: `fetch('/api/documents/${id}', { method: 'DELETE' })`
   - On success: `router.refresh()` to re-fetch page data (Next.js App Router pattern)
   - On error: show alert with error message from response
6. Empty state: `<p className="text-sm text-muted-foreground">No documents attached yet.</p>`
7. Style: match the tag-editor component — rounded-md, bg-card, border, text-sm, text-muted-foreground for secondary text. Use `space-y-2` for list spacing.

Implementation:

3. File input: `accept=".pdf,.doc,.docx,.txt,.png,.jpg,.jpeg"` — client-side hint only, server validates via magic bytes

   f. On 413: set error "File must be under 10MB"
   g. On other error: set error "Upload failed. Please try again."
   h. Finally: setUploading(false)
6. Show loading state: disable submit button, show "Uploading..." text
7. Style: match existing form patterns. Use `bg-card`, `rounded-md`, `border`, `p-4` layout. Labels with `text-sm font-medium`. Input fields with `border rounded-md px-3 py-1.5 text-sm w-full`.
