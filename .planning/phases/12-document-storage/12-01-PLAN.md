Implement four functions, all returning `Result<T, AppError>`:

1. `listDocuments(userId: UserId, applicationId: ApplicationId): Promise<Result<Document[], AppError>>`
   - Use `withRls(userId, ...)` to query `tx.document.findMany({ where: { applicationId }, orderBy: { createdAt: 'desc' } })`

   - Verify `document.application.userId === Number(userId)`, return err(notFound) if mismatch

3. `uploadDocument(userId: UserId, applicationId: ApplicationId, file: File, kind: DocumentKind, notes?: string): Promise<Result<{ documentId: DocumentId; eventId: EventId }, AppError>>`
   - Validate file size: `if (file.size > MAX_FILE_SIZE) return err(validation(...))`
   - Validate MIME via magic bytes: read first 8 bytes of the file using `file.slice(0, 8).arrayBuffer()`, check against known magic-byte signatures. Reject if MIME not in ALLOWED_MIME_TYPES.
   - Use `withRls(userId, async (tx) => { ... })`:
     a. Verify application belongs to user
     b. Create Document row (id auto-generated)
     c. Build storage path: `path.join(process.cwd(), 'data', 'documents', String(applicationId), String(document.id), sanitizedFilename)`
     d. Sanitize filename: replace `[^a-zA-Z0-9._-]` with `_`, limit to 200 chars
     e. Ensure directory exists: `await fs.mkdir(path.dirname(storagePath), { recursive: true })`
     f. Write file: `await fs.writeFile(storagePath, Buffer.from(await file.arrayBuffer()))`
     g. Update Document row with storagePath
     h. Create Event(type='document_uploaded', data={documentId, filename, kind}, source='manual')
     i. Return { documentId, eventId }
   - On any error after file write, attempt cleanup: `await fs.rm(storagePath, { force: true }).catch(() => {})`

4. `deleteDocument(userId: UserId, documentId: DocumentId): Promise<Result<void, AppError>>`
   - Use `withRls(userId, async (tx) => { ... })`:
     a. Find document with application userId check (same pattern as getDocument)
     b. Delete Event rows referencing this document in data (optional — cascade handles it, but explicit is safer)
     c. Delete Document row
     d. Delete file from disk: `await fs.rm(document.storagePath, { force: true })`

**Magic byte check implementation:** Use the `file-type` package if available in node_modules (check with `ls node_modules/file-type` first). If not available, implement a minimal helper:
```typescript
async function detectMimeType(buffer: ArrayBuffer): Promise<string | null> {
  const bytes = new Uint8Array(buffer)
  // PDF: %PDF
  if (bytes[0] === 0x25 && bytes[1] === 0x50 && bytes[2] === 0x44 && bytes[3] === 0x46) return 'application/pdf'
  // PNG: \x89PNG
  if (bytes[0] === 0x89 && bytes[1] === 0x50 && bytes[2] === 0x4E && bytes[3] === 0x47) return 'image/png'
  // JPEG: \xFF\xD8\xFF
  if (bytes[0] === 0xFF && bytes[1] === 0xD8 && bytes[2] === 0xFF) return 'image/jpeg'
  // ZIP-based (docx, xlsx): PK\x03\x04
  if (bytes[0] === 0x50 && bytes[1] === 0x4B && bytes[2] === 0x03 && bytes[3] === 0x04)
    return 'application/vnd.openxmlformats-officedocument.wordprocessingml.document'
  // OLE2 (doc): \xD0\xCF\x11\xE0
  if (bytes[0] === 0xD0 && bytes[1] === 0xCF && bytes[2] === 0x11 && bytes[3] === 0xE0) return 'application/msword'
  // Plain text: check if all bytes are printable ASCII (fallback)
  if (bytes.every(b => (b >= 0x20 && b <= 0x7E) || b === 0x09 || b === 0x0A || b === 0x0D)) return 'text/plain'
  return null
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API | User-supplied file content crosses here (untrusted binary data) |
| API → filesystem | File written to disk — path traversal risk |
| API → DB | Document metadata stored — must validate all fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Tampering | File upload | mitigate | Validate MIME via magic bytes (not extension); reject mismatched types |
| T-12-02 | Information Disclosure | File storage path | mitigate | Server-generated path under data/documents/{appId}/{docId}/; sanitized filename; no user-controlled path components |
| T-12-03 | Denial of Service | File upload size | mitigate | 10MB hard limit checked before buffering full file into memory |
| T-12-04 | Elevation of Privilege | Document access | mitigate | withRls transaction + application.userId check on every get/delete; RLS policy on documents table |
| T-12-05 | Spoofing | Filename in Content-Disposition | mitigate | Sanitize filename (replace special chars); quote in header to prevent injection |
| T-12-06 | Tampering | Path traversal | mitigate | Sanitize filename: replace `[^a-zA-Z0-9._-]` with `_`; build path from server-generated IDs only |
</threat_model>
