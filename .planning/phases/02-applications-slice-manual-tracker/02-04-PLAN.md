       - `addStageAction(applicationId: number, prev, formData)`: a curried action — Plan 04 components must call `addStageAction.bind(null, app.id)` to inject applicationId. Parses `stageInputSchema` → requireUser → `addStage(userId, ApplicationId(String(applicationId)), parsed.data)` → revalidatePath, return initialOk
       - `updateStageAction(stageId: number, applicationId: number, prev, formData)`: curried with stageId + applicationId. Parses `stageInputSchema.partial()` → requireUser → `updateStage(userId, StageId(String(stageId)), parsed.data)` → revalidatePath, return initialOk
       - `completeStageAction(stageId: number, applicationId: number, prev, formData)`: curried. Parses `stageOutcomeEnum` from formData.get('outcome') → requireUser → `completeStage(userId, StageId(String(stageId)), outcome)` → on Err: NotFound → 'Stage not found.', Conflict → 'Stage already completed.', else → 'Could not complete stage.' → revalidatePath, return initialOk
       - `updateNotesAction(applicationId: number, prev, formData)`: curried with applicationId. Parses `notesInputSchema` → requireUser → `updateApplicationNotes(userId, ApplicationId(String(applicationId)), parsed.data)` → on Err: NotFound → 'Foray not found.', else → 'Could not save notes.' → if value.notesChanged revalidatePath, return initialOk

| Boundary | Description |

| Page → query | Page passes `userId` from `requireUser()`; query enforces tenancy via `withRls`. |
| Component → DOM | React auto-escapes user-rendered strings; no `dangerouslySetInnerHTML` allowed in any Plan 04 component. |

|-----------|----------|-----------|-------------|-----------------|

| T-02-04-03 | Tampering | XSS via notes / JD / company name | mitigate | React auto-escapes all string props. NO `dangerouslySetInnerHTML` allowed (verified by grep in Task 2 acceptance). Notes/JD stored as raw text, rendered via `{notes}` interpolation. |
