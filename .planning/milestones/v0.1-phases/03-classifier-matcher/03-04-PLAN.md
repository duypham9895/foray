<objective>
Build the classifier-fixtures suite — anonymized real-shape email samples that act as the regression fence for the rules.ts regex tier table. Each fixture is a JSON file with subject, body excerpt, and expected label. The harness test loads ALL fixtures and runs each through `classifyByRules` (Plan 03-01) — asserting the label matches.

This plan does NOT exercise the LLM path. The LLM is mocked in Plan 03-02's tests; running real fixtures through real Anthropic in CI is forbidden (cost + flakiness). The fixtures verify the rule layer ONLY. When a future fixture fails because rules.ts misses a phrasing, that's a rule-tuning task — not an LLM tuning task.

Purpose: when the owner reports a misclassification ("foray's classifier just labeled my recruiter outreach as a rejection"), the fix is:
1. Add the offending email (anonymized) to the appropriate `should-not-have-fired/` or label subdir.
2. Run the test — it fails.
3. Tune `rules.ts` regex(es) until the test passes.
4. Commit both files together.

This is the loop that prevents Pitfall #4 (first wrong classification destroys trust). Real-shape samples are the truth; rules are derived from them.

Output:
- ≥8 JSON fixture files across 6 subdirs
- README.md documenting the fixture contract
- classifier-fixtures.test.ts harness running all fixtures through classifyByRules
</objective>

**classifier-fixtures.test.ts:**
- enumerates all `.json` files under `tests/integration/classifier-fixtures/` (recursively, excluding README.md)
- for each fixture: parses JSON, runs `classifyByRules({subject, bodyExcerpt})`, asserts `result.label === fixture.expectedLabel`
- DOES NOT assert on `result.confidence` strictly — the rule's tier may not match expectedConfidence exactly (e.g., the workday-sample fires at 0.80 even though the fixture's notes say "tricky"). The test asserts label only.
- Uses Vitest's `it.each` (or a `describe` per file) so each fixture is its own named test in the output — when a fixture fails, the test name names the file path, not just "fixture #5".

**Test structure:**
```typescript
import { describe, it, expect } from 'vitest'
import { readdirSync, readFileSync } from 'node:fs'
import path from 'node:path'
import { classifyByRules } from '@/features/classifier/rules'

const FIXTURES_ROOT = path.join(__dirname, 'classifier-fixtures')

function loadAllFixtures(): Array<{
  filepath: string
  subdir: string
  filename: string
  fixture: { subject: string; bodyExcerpt: string; expectedLabel: string; source: string }
}> {
  const subdirs = readdirSync(FIXTURES_ROOT, { withFileTypes: true })
    .filter(d => d.isDirectory())
    .map(d => d.name)
  return subdirs.flatMap(subdir => {
    const subdirPath = path.join(FIXTURES_ROOT, subdir)
    return readdirSync(subdirPath)
      .filter(f => f.endsWith('.json'))
      .map(filename => {
        const filepath = path.join(subdirPath, filename)
        const content = readFileSync(filepath, 'utf8')
        const fixture = JSON.parse(content) as { subject: string; bodyExcerpt: string; expectedLabel: string; source: string }
        return { filepath, subdir, filename, fixture }
      })
  })
}

describe('classifier rule fixtures', () => {
  const fixtures = loadAllFixtures()

  it('discovers ≥8 fixtures across ≥5 subdirs', () => {
    expect(fixtures.length).toBeGreaterThanOrEqual(8)
    const subdirs = new Set(fixtures.map(f => f.subdir))
    expect(subdirs.size).toBeGreaterThanOrEqual(5)
  })

  it.each(fixtures)('$subdir/$filename → expectedLabel matches', ({ filepath, fixture }) => {
    const result = classifyByRules({ subject: fixture.subject, bodyExcerpt: fixture.bodyExcerpt })
    expect(result.label, `Fixture ${filepath} expected ${fixture.expectedLabel} but got ${result.label} (confidence ${result.confidence})`).toBe(fixture.expectedLabel)
  })

  it('every fixture has the required keys', () => {
    for (const { filepath, fixture } of fixtures) {
      expect(fixture, `Fixture ${filepath} is missing fields`).toMatchObject({
        subject: expect.any(String),
        bodyExcerpt: expect.any(String),
        expectedLabel: expect.stringMatching(/^(rejection|interview_invite|recruiter_outreach|noise|unmatched)$/),
        source: expect.stringMatching(/^(real|synthetic)$/),
      })
    }
  })

  it('every fixture body excerpt is ≤500 chars (mirrors production privacy rule)', () => {
    for (const { filepath, fixture } of fixtures) {
      expect(fixture.bodyExcerpt.length, `Fixture ${filepath} body exceeds 500 chars`).toBeLessThanOrEqual(500)
    }
  })
})
```

This pattern means:
- 1 "discovers" test
- N parametrized "label matches" tests (one per fixture)
- 1 "shape valid" test
- 1 "≤500 chars" test
- Total: N + 3 tests, where N ≥ 8.
